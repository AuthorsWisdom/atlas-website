# XAtlas Electron Build Log — 2026-03-27

## Mission
Convert XAtlas from a web app (Next.js + Supabase + Fly.io) into a standalone Mac desktop app using Electron, with the Python backend bundled as a silent sidecar binary.

---

## What Was Built

### 1. PyInstaller Backend Binary
- Bundled `~/projects/atlas-backend/main.py` into a standalone 56MB Mac binary (`dist/xatlas-backend`)
- Added `if __name__ == "__main__": uvicorn.run(app, host="0.0.0.0", port=8080)` entry point
- Required Python 3.12 (system had 3.9 which doesn't support `type | None` syntax)
- Created a `.venv` with Python 3.12 via Homebrew and installed all 21+ dependencies
- Binary starts uvicorn on port 8080, serves FastAPI backend silently

### 2. Electron Shell
- **electron/main.js** — Main process that:
  - Loads `.env` from `atlas-backend/.env` and `.env.local` and passes vars to backend process
  - Spawns the PyInstaller binary as a silent sidecar (no terminal window)
  - Polls port 8080 every 500ms until backend responds
  - Creates BrowserWindow (1400x900, hiddenInset title bar for native Mac look)
  - Loads `http://localhost:3000` (Next.js dev server)
  - Kills backend on app quit
  - IPC handlers for encrypted key storage (`get-key`, `set-key`, `delete-key`)
  - IPC handlers for generic data persistence (`get-data`, `set-data`) — portfolio, watchlist
  - Seeds AI provider keys from `.env` into Electron store on first launch
- **electron/preload.js** — contextBridge exposing `window.electron` API to renderer
- **electron/icons/** — Generated 1024x1024 icon (dark bg, white X, "ATLAS" text) + .icns via iconutil

### 3. Build Configuration

## Architecture Decisions

1. **Electron over Tauri** — Project already had a Tauri setup but the user specifically requested Electron. Electron adds ~116MB (framework download) but provides Node.js ecosystem compatibility.

2. **PyInstaller --onefile** — Single binary (~56MB) instead of directory mode. Simpler to bundle as Electron resource, slower cold start (unpacks to temp dir) but acceptable.

3. **Port 8080 for backend** — Changed from 8000 to avoid conflicts. Hardcoded, not configurable.

4. **Finnhub for everything** — Stocks (WebSocket + REST) and crypto (REST polling) all through Finnhub. Single API key, no IP restrictions, free tier adequate for local use.

5. **Electron store for persistence** — API keys encrypted, portfolio/watchlist stored locally. No cloud dependency. Survives app restarts.

6. **No auth, no tiers** — Everything unlocked. `isPro = true` hardcoded. No sign-in required. Local desktop app has no need for user management.

7. **Next.js dev server in Electron** — Dev mode runs `next dev` + Electron. Production build bundles `.next/` output. This means the app requires Node.js runtime even in production — a tradeoff for rapid development.

8. **Backend .env keys seeded to Electron store** — On first launch, `ANTHROPIC_API_KEY` → `anthropic`, `CHATGPT_API_KEY` → `openai`, `GROK_API_KEY` → `xai` automatically populate the AI Council provider keys.
